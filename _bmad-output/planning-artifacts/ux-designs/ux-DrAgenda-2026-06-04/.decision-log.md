# Decision Log — UX DrAgenda
Criado em: 2026-06-04 | Sessão com Rodrigo Navarro

---

| ID | Decisão | Escolha | Racional |
|----|---------|---------|---------|
| UX-D-01 | Tom visual | Clean/Minimalista | Interface que "desaparece" — alinha com promessa do produto ("2 toques, sem treinamento") |
| UX-D-02 | Cor de marca | Teal `#0D9488` | Distintivo no setor saúde (dominado por azuis), contraste adequado, não conflita com as 6 cores de status |
| UX-D-03 | Logo / wordmark | DM Sans Light+Medium, wordmark | Geométrica, amigável, humanizada. "Dr" light + "Agenda" medium. Sem ativo gráfico pré-existente. |
| UX-D-04 | Tema claro/escuro | System default (`prefers-color-scheme`) | Respeita contexto de cada usuário; Wanessa beneficia em ambiente com luz baixa. Ambos os temas implementados desde o MVP. |
| UX-D-05 | Acessibilidade | WCAG 2.1 AA + daltonismo | Os 6 status usam cores; regra inviolável: cor + ícone + rótulo em todo indicador de status. Custo de design baixo. |
| UX-D-06 | Modelo de navegação | Bottom Tab Bar | Favorece usuárias diárias (Maria e Wanessa). Cada perfil vê somente as abas relevantes. |
| UX-D-07 | Estrutura de abas | Staff: Agenda/Buscar/Clientes/Menu; Profissional: Agenda/Menu; Admin: Agenda/Configurações/Menu | Cada perfil vê somente o que usa. |
| UX-D-08 | Botão novo agendamento | FAB flutuante | Libera uma aba. Visível para Staff e Profissional. Para Wanessa, campo "Profissional" pré-preenchido e bloqueado. |
| UX-D-09 | Card de agendamento | Card Médio (data+hora+nome+tipo+botão de ação visível) | Alinha com regra dos 2 toques — botão de ação primário visível sem abrir o card. ~3-4 cards por tela. |
| UX-D-10 | Formulário de novo agendamento | Bottom Sheet (mobile) / Drawer lateral (desktop) | Contexto da agenda permanece visível ao fundo. Maria pode verificar conflitos enquanto preenche. |
| UX-D-11 | Feedback de validação | Combinação: erros inline + toasts de sucesso | Erros inline identificam o campo exato. Toasts confirmam ações concluídas com feedback positivo. |
| UX-D-12 | Dialog de double-booking | Dialog bloqueante centralizado | A interrupção é desejável — double-booking é exceção intencional. Dialog comunica a gravidade da sobreposição. |
| UX-D-13 | Navegação da agenda | Toggle Dia/Semana + swipe horizontal | Toggle dá descobribilidade. Swipe dá fluidez na navegação temporal. Padrão consagrado por apps de calendário. |
| UX-D-14 | Busca de próximo disponível | Lista agrupada por data + chips de filtro | Chips de filtro completam FR-029 (filtros opcionais por profissional e tipo). 1 toque, resultado imediato. |
| UX-D-15 | Tela pública de autoatendimento do paciente | Confirmação/Cancelamento + branding + sugestão de reagendamento. Botão confirmar primário (brand-500, cheio); cancelar secundário (outline).
